Сценарій 1:
Реєстрація користувача:
-генерується новий id;
-додається нове ім'я;
-додається нове прізвише;
-додається логін;
-додається пошта;
-встановлюється роль(basic, cc(content creator));
-встановлюється хеш пароля.

Сценарій 2:
Вхід користувача:
-генерується токен та записується в базу данних;

Сценарій 3:
Додавання пісні:
-генерується новий id;
-вказується альбом(якщо відноситься);
-вказується автор(якщо авторів декілька: додаються їх id в массив; якщо не існує: виконується Сценарій 4);
-вказується категорія(якщо не існує: виконується Сценарій 5);
-додається назва;
-додається текст пісні(якщо доступний);
-програмно визначається довжина треку та додається до бази даних;
-додається обкладинка(якщо доступна);
-додається рік випуску пісні;
-записується дата додавання треку до бази даних;
-записується шлях зберігання треку.

Сценарій 4:
Додавання автору:
-генерується новий id;
-вказується назву автора.

Сценарій 5:
Додавання жанру:
-генерується новий id;
-вказується назва жанру.

Сценарій 6:
Додавання альбому:
-генерується новий id;
-вказується автор(якщо авторів декілька: додаються їх id в массив; якщо не існує: виконується Сценарій 4);
-вказується назва альбому;
-додається обкладинка(якщо доступна).

Сценарій 7:
Створення плейлиста:
-генерується новий id;
-вказується id користувача;
-вказується назва альбому;
-встановлюється дата зміни.

Сценарій 8:
Додавання пісень до плейлиста:
-визначаеться id плейлиста;
-вказується id пісні та додається до масиву треків.

Сценарій 9:
Видалення пісень з плейлиста:
-визначається id пісні та плейлиста;
-видаляється запис в масиву.


Сценарій 10:
Видалення плейлиста:
-визначається id плейлиста;
-видаляється запис в таблиці.

Сценарій 11:
Створення кімнати:
-генерується id кімнати;
-вказується id власника;
-вказується назва кімнати.

Сценарій 12:
Вхід до кімнати:
-додається id користувача в масив відвідувачів.

Сценарій 13:
Видалення кімнати:
-Визначаеться id кімнати та видаляеться запис.
